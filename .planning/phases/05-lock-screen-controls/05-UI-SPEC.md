## Default Cover Placeholder ŌĆö New Asset

Phase 5 requires one new file: `/public/images/default-cover.svg`

This SVG is referenced by `navigator.mediaSession.metadata` when `cover_url` is null (Decision D-03). It is served at a stable URL so the OS can fetch it for the lock screen. It is NOT rendered inline in the browser UI.

**Specification:**

| Property | Value |
|----------|-------|
| File path | `/public/images/default-cover.svg` |
| Dimensions | Square ŌĆö viewBox="0 0 512 512" |
| Background fill | #1e293b (var(--color-placeholder-bg)) |
| Icon color | #94a3b8 (var(--color-text-muted)) |
| Icon | Open book / headphones motif ŌĆö simple, single-path geometric icon |
| Format | Inline SVG, no external references, no scripts |
| Usage context | Media Session API artwork src only ŌĆö not in DOM |

**Design intent:** Matches the existing `.cover-placeholder` background color so the lock screen shows a polished, branded placeholder rather than a default OS music icon. The icon must be recognizable at both 96px and 512px (the two sizes declared in Media Session artwork ŌĆö Decision D-02).

## Media Session API Contract

This section defines the JavaScript interaction contract. The "UI" for this phase is the OS lock screen ŌĆö not the browser DOM.

### Metadata fields

| Field | Value |
|-------|-------|
| `title` | `"{Book Title} -- Ch. {N}: {Chapter Name}"` (Decision D-08) |
| `artist` | Book author string |
| `album` | Not set (leave empty string) |
| `artwork[0]` | `{ src: cover_url ?? '/images/default-cover.svg', sizes: '96x96', type: 'image/jpeg' }` |
| `artwork[1]` | `{ src: cover_url ?? '/images/default-cover.svg', sizes: '512x512', type: 'image/jpeg' }` |

When `cover_url` is null, both artwork entries use `/images/default-cover.svg` with `type: 'image/svg+xml'`.

### Position state

| Property | Value |
|----------|-------|
| `duration` | Chapter `end_sec - start_sec` (chapter duration, not full book) |
| `playbackRate` | `window._spineAudio.playbackRate` (current speed) |
| `position` | `window._spineAudio.currentTime - chapter.start_sec` (chapter-relative offset) |

Scrubber scope: current chapter only (0:00 to chapter duration). Decision D-04.

Update interval: 1 second via `setInterval`. Decision D-05.

### Action handlers

| Action | Behavior |
|--------|----------|
| `play` | Already registered in Phase 4 ŌĆö no change |
| `pause` | Already registered in Phase 4 ŌĆö no change |
| `nexttrack` | Already registered in Phase 4 ŌĆö no change |
| `previoustrack` | Already registered in Phase 4 ŌĆö no change |
| `seekto` | New in Phase 5 ŌĆö maps `details.seekTime` (chapter-relative) to absolute audio position via `chapter.start_sec + details.seekTime` |

### Metadata lifecycle

| Event | Action |
|-------|--------|
| New book starts playing | Set MediaMetadata with book info + chapter 0 |
| Chapter changes | Update `title` field to reflect new chapter number and name |
| Audio pauses | Keep metadata visible ŌĆö do NOT clear (Decision D-09) |
| User logs out | Clear `navigator.mediaSession.metadata = null` |
| Player closed (book set to null) | Clear `navigator.mediaSession.metadata = null` |

## Implementation Constraints

Derived from CONTEXT.md decisions and CLAUDE.md conventions:

1. **No new CSS classes required.** The player bar UI (existing) is unchanged. No new visible components are added to `public/style.css`.

2. **No new HTML structure required.** All changes are JavaScript logic in `public/index.html` Alpine stores and handlers.

3. **One new static file:** `/public/images/default-cover.svg` ŌĆö must be served by the existing `serveStatic` middleware. The `/public/images/` directory must be created.

4. **SVG must be self-contained.** No external URLs, no `<script>` tags, no `<use>` with external hrefs. The OS fetches this file independently.

5. **Guard pattern:** All Media Session calls wrapped in `if ('mediaSession' in navigator)` ŌĆö matching established Phase 4 pattern.

6. **Bun runtime.** No new npm packages. All Media Session work uses Web APIs only.
