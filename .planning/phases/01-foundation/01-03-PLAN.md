   - Export type `FallbackMetadataJson = { title?: string; author?: string; narrator?: string; year?: string; description?: string; genre?: string; publisher?: string; language?: string; series_title?: string; series_position?: string; }`
   - Export `applyFallbackMetadata(metadata: NormalizedMetadata, m4bPath: string): NormalizedMetadata`
     - Compute `dir = path.dirname(m4bPath)`
     - Check for `metadata.json` at `path.join(dir, 'metadata.json')` — if exists, read and JSON.parse it
     - For each field in FallbackMetadataJson: if the corresponding field in `metadata` is null and the JSON field is non-empty, set it (per D-07: embedded wins, fallback fills gaps)

2. Create `src/scanner/index.ts`:
   - Export `scanFile(db: Database.Database, filePath: string): Promise<void>`
     - `fs.statSync(filePath)` to get mtime and size
     - Query `SELECT file_mtime, file_size FROM books WHERE file_path = ?` — if match, return early (incremental skip per D-02)
     - Call `probeFile(filePath)` and `normalizeMetadata(result)`
     - Call `applyFallbackMetadata(metadata, filePath)` to fill gaps (SCAN-05)
     - Determine cover: if `metadata.has_cover_stream`, call `extractCoverArt(filePath, true)` to get coverPath. Otherwise call `resolveCoverPath(filePath, false)` for D-10 fallback.
     - UPSERT into books table using prepared statement (all fields from NormalizedMetadata + file_mtime, file_size, cover_path). Use `INSERT ... ON CONFLICT(file_path) DO UPDATE SET ...` pattern. Set `is_missing = 0` in both insert and update (handles D-04 reappearance).
     - Delete existing chapters for this book_id, then INSERT new chapters from metadata.chapters
     - Wrap chapter delete + insert in a transaction for atomicity
     - On error (ffprobe failure, bad file), log warning with file path and error message, skip this file, do not stop the scan (per Pitfall 5 from research)

   - Export `scanLibrary(db: Database.Database, libraryRoot: string): Promise<void>`
     - Call `walkLibrary(libraryRoot)` to get all .m4b paths
     - Log: `Scanning ${paths.length} .m4b files in ${libraryRoot}`
     - Process files with concurrency limit of 4 (use a simple semaphore pattern: maintain a Set of active promises, when Set.size >= 4, await Promise.race on the set)
     - After all files processed, mark missing files:
       ```sql
       UPDATE books SET is_missing = 1, updated_at = datetime('now')
       WHERE file_path NOT IN (${placeholders}) AND is_missing = 0
       ```
       (where placeholders is the list of paths found in current walk)
     - Log summary: `Scan complete: ${paths.length} files found, ${newCount} new/updated, ${missingCount} marked missing`

1. Create `src/scanner/watcher.ts`:
   - Export `startWatcher(db: Database.Database, libraryRoot: string): void`
   - Use a `setInterval` polling approach (simpler and more reliable in Docker than chokidar — per research, chokidar v5 ESM compatibility with Bun is an open question and setInterval is equally correct for this use case)
   - Set interval to 5 minutes (300_000 ms) by default, configurable via `SCAN_INTERVAL_MS` env var
   - Each interval tick calls `scanLibrary(db, libraryRoot)` — this handles adds, changes, and removals because scanLibrary walks, diffs, and marks missing
   - Log on each re-scan: `Re-scanning library at ${new Date().toISOString()}`
   - Export `stopWatcher(): void` that calls `clearInterval` — needed for clean shutdown and testing

   Decision note: Using setInterval over chokidar per Claude's discretion. Rationale: chokidar requires `usePolling: true` in Docker anyway (which is functionally identical to setInterval), adds a dependency, and has an unverified ESM+Bun compatibility story. setInterval is zero-dependency and 100% reliable.
