## Interaction Contract

### Create User Form

- Form is **always visible** above the table — not toggled (Claude's discretion: primary admin action, toggle adds complexity with no benefit on an admin-only page).
- Layout on desktop (>= 640px): single row — `[Username input] [Password input] [Role select] [Create User button]` with `gap: 8px`, flex-wrap on mobile.
- Layout on mobile (< 640px): stacked fields, full-width inputs and button.
- Role dropdown options: "user" (default, selected), "admin".
- On submit: button shows "Creating..." (disabled) during fetch. On success: form fields clear, `statusMsg` sets to "User {username} created", auto-clears after 3 seconds.
- On error: inline error below form (`.admin-form-error`), text "Username already exists. Choose a different username." or "Failed to create user. Try again."
- Empty field validation: browser-native `required` attribute on username and password fields. No custom error for this.

### Delete Action (D-11, D-12, D-13)

- Default state: "Delete" button (`.btn-delete`) at end of each row.
- Own row (`user.username === $store.auth.username`): no delete button rendered.
- Last-admin row: delete button rendered but `.btn-delete-disabled`, `disabled` attribute set, `title="Cannot delete the last admin"`.
- First click: button text changes to "Confirm?" (`.btn-delete-confirm`), border becomes `--color-error`, 3-second timeout before reverting. No modal, no overlay.
- Confirm click: button becomes disabled, text "Deleting...", row removed on success, `statusMsg` sets to "User {username} deleted", auto-clears after 3 seconds.
- On error: `statusMsg` shows "Failed to delete user. Try again." in error color.

### Reset Password Action (D-14, D-15)

- Default state: "Reset Password" button (`.btn-action`) in Actions column.
- Click: Actions column content replaces with inline password input + "Save Password" button + "Keep Password" button. The Reset Password button disappears.
- Input: `type="password"`, placeholder "New password", `.admin-reset-input` (44px height, full width of Actions cell).
- Save Password click: "Save Password" becomes "Saving..." (disabled). On success: form collapses back to "Reset Password" button, `statusMsg` sets to "Password reset for {username}".
- Keep Password click: collapses back to "Reset Password" button with no status message.
- On error: inline error text in cell "Reset failed. Try again."

### Status Message

- Single `.admin-status` bar below the page heading, above the create form.
- Hidden (`x-show="statusMsg"`) when empty.
- Background: `var(--color-surface)`, text: `var(--color-text)`, padding: 8px 16px, border-radius 6px, font-size 16px.
- Auto-clears after 3 seconds via `setTimeout`.
- Shows for both success and error at page level (cell-level errors shown inline in the relevant row/form instead).

### "(You)" Badge (D-05)

- Renders as `.admin-badge-you` pill immediately after the username text, inline in the Username cell.
- Text: "(You)" — parentheses included.
- Never renders on rows other than the current user.

### Date Formatting (Claude's Discretion)

- `created_at`: `new Date(str).toLocaleDateString(undefined, { year: 'numeric', month: 'short', day: 'numeric' })` — e.g., "Mar 23, 2026".
- `last_login_at`: `last_login_at ? new Date(last_login_at).toLocaleString(undefined, { month: 'short', day: 'numeric', year: 'numeric', hour: '2-digit', minute: '2-digit' }) : 'Never'`.
- Both use locale-appropriate format. No external date library.

### Nav Bar Extension (D-01, D-03)

- Add `|| $store.app.view === 'admin'` to the nav bar's `x-show` condition.
- Add `<a class="nav-admin-link" x-show="$store.auth.role === 'admin'" @click.prevent="$store.app.view = 'admin'" href="#">Users</a>` in `.nav-right`, before the logout button.
- `.nav-admin-link`: matches `.nav-logout` styling (14px, `--color-text-muted`), `cursor: pointer`, hover: `--color-text`.
- When on admin view, the "Users" link is active; visually distinguish with `color: var(--color-text)` and `font-weight: 600` using `:class` binding.
- Player bar remains visible on admin view (D-03) — no change needed to player z-index or visibility.

### Loading State

- On `x-init="loadUsers()"`: while `loading === true`, show a single-row table placeholder — one `<tr>` spanning all columns, centered text "Loading users..." in `--color-text-muted`, 16px.
- No skeleton animation required (admin page, low frequency, fast load).

## Accessibility Contract

- All `<input>` elements have associated `<label>` elements (even if visually hidden via `sr-only` class) or `aria-label` attributes.
- Disabled delete buttons use the native `disabled` attribute (not just visual styling).
- The `title` attribute on disabled delete buttons (`"Cannot delete the last admin"`) provides tooltip text for pointer users; add `aria-label` with the same text for screen reader users.
- Confirm-state delete button: `aria-pressed="true"` when in confirm mode.
- Status message bar: `role="status"` `aria-live="polite"` so screen readers announce it without interrupting.
- Table: `<thead>` with `<th scope="col">` for each column.
- Inline password reset input: `aria-label="New password for {username}"` (constructed dynamically from Alpine).
